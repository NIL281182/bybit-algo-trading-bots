# Инструкция по запуску бота Donchian v3

## 1. Получение API-ключей Bybit Testnet

### 1.1 Регистрация
- Перейди на https://testnet.bybit.com
- Нажми **Sign Up** (регистрация бесплатная)

### 1.2 Создание API-ключа
- Аватар → **API Keys** → **Create New Key**
- Тип: **System-generated API Keys**
- Название: `donchian_v3`
- Permissions: ✅ Read, ✅ Write, ❌ Withdraw (обязательно сними!)
- IP Restriction: **No restriction** (для теста)
- Нажми Submit, введи 2FA

### 1.3 Сохранение
- **API Key** — скопируй
- **API Secret** — скопируй (**показывается только один раз!**)

### 1.4 Тестовый депозит
- В личном кабинете найди **Assets** / **Faucet** / "Get Test USDT"
- Запроси 50 000 USDT

## 3. Настройка бота

### Способ А: Файл `.env` (рекомендуется)

1. Скопируй `.env.example` в `.env`:
   ```bash
   cp .env.example .env
   ```
2. Открой `.env` и вставь свои ключи:
   ```
   BYBIT_API_KEY=your_api_key_here
   BYBIT_API_SECRET=your_api_secret_here
   ```
3. Запусти бота:
   ```bash
   python bot_donchian_v3.py
   ```

Файл `.env` уже добавлен в `.gitignore` — он **не попадёт** в систему контроля версий.

### Способ Б: Переменные окружения

**Windows (PowerShell):**
```powershell
$env:BYBIT_API_KEY="твой_api_key"
$env:BYBIT_API_SECRET="твой_api_secret"
python bot_donchian_v3.py
```

**Bash/Git Bash:**
```bash
export BYBIT_API_KEY="твой_api_key"
export BYBIT_API_SECRET="твой_api_secret"
python bot_donchian_v3.py
```

## 5. Быстрый запуск через `.bat` (Windows)

Для удобства созданы два файла в папке проекта:

### `start_bots.bat` — запуск обоих ботов

Двойной клик по файлу откроет **два независимых CMD-окна**:
- **BTC Donchian v3 Bot** — `bot_donchian_v3.py`
- **ETH EMA Pullback Bot** — `bot_ema_pullback_eth.py`

Каждое окно работает автономно. Telegram-алерты активируются автоматически, если `.env` заполнен.

### `stop_bots.bat` — остановка ботов

Двойной клик:
1. Закрывает окна по заголовкам (мягкая остановка)
2. Спрашивает, нужно ли принудительно убить все процессы `python.exe`

**Важно:** файлы ищут `.env` в той же папке, где лежат сами. Не перемещай `.bat` в другие директории.

## 6. Что делает бот

1. **Раз в час** загружает дневные свечи BTCUSDT с Bybit
2. **Если нет позиции:**
   - Проверяет: `close > 20-дневный high` И `close > EMA50`
   - Если сигнал есть — рыночный вход + стоп-лосс (entry − 2×ATR)
   - Риск: 1% от equity
3. **Если есть позиция:**
   - Проверяет: не закрылась ли свеча ниже EMA20
   - Если да — рыночный выход
   - Если стоп сработал на бирже — бот увидит пропажу позиции
4. **Всё логируется** в `bot_donchian_v3.log`

## ⚠️ Важные предупреждения

- **Только Testnet** — боты используют Testnet по умолчанию. Для реального счёта нужно изменить `testnet=True` на `False` и получить ключи с `bybit.com`.
- **Withdraw = False** — при создании API-ключа обязательно сними галочку с Withdraw. Боту не нужен вывод средств.
- **Плечо** — в коде `LEVERAGE = 2`. На Testnet можно тестировать разные значения.
- **Не запускай на реальные деньги** без недели тестов на Testnet.
- **`.env` не должен попадать в git** — файл `.env` с ключами добавлен в `.gitignore`. Если вы используете Git, проверьте `git status` — `.env` не должен отображаться как untracked.
- **Общий код** — `bybit_utils.py` содержит общую логику для обоих ботов. Изменения в нём влияют на оба бота сразу.

## 8. Параметры (настройки в коде)

| Параметр | Значение | Описание |
|----------|----------|----------|
| `LEVERAGE` | 2 | Плечо |
| `RISK_PCT` | 0.01 | Риск на сделку (1%) |
| `DONCHIAN` | 20 | Длина канала Дончиана |
| `EMA_FILT` | 50 | EMA фильтр тренда |
| `EMA_TRAIL` | 20 | EMA для трейлинг-выхода |
| `ATR_MUL` | 2.0 | Множитель ATR для стопа |
| `CHECK_INTERVAL` | 3600 | Проверка раз в час (сек) |

Боты отправляют уведомления в Telegram при сделках и ошибках.

**Подробная инструкция** — см. [`TELEGRAM_SETUP.md`](TELEGRAM_SETUP.md):

- Получение Chat ID (личный / групповой)
- Тестовая отправка сообщения вручную
- Разбор 5 типичных ошибок

Кратко:
1. @BotFather → `/newbot` → получи **BOT_TOKEN**
2. @userinfobot → получи **CHAT_ID**
3. Запиши в `.env`
4. Протестируй тестовым скриптом из `TELEGRAM_SETUP.md`

Бот будет отправлять:
- 🚀 Вход в позицию (цена, стоп, qty, equity)
- 📉 Выход из позиции (причина)
- ⛔ Стоп-лосс сработал
- ⚠️ Критические ошибки
- 🤖 Запуск бота

## 10. Общий модуль `bybit_utils.py`

Вынесены общие функции для обоих ботов:

| Функция | Назначение |
|---------|------------|
| `get_trade_session` / `get_data_session` | Подключение к API (Testnet / Mainnet) |
| `fetch_klines` | Загрузка свечей с фильтром выбросов |
| `get_wallet_balance` | Доступный баланс |
| `get_position` | Проверка открытой позиции |
| `get_instrument_info` | Информация о торговом инструменте (мин. лот и т.д.) |
| `set_leverage` | Установка плеча |
| `place_market_order` | Выставление рыночного ордера + стоп-лосс |
| `close_position_market` | Закрытие позиции |
| `calculate_qty` | Расчёт размера позиции по риску (1%) |
| `normalize_qty` | Округление qty до допустимого шага и минимума |

**Важно:** расчёт `calculate_qty` теперь корректен — **плечо не участвует** в формуле qty. Плечо только снижает требуемый margin.
